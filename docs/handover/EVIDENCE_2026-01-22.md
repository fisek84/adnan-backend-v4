## Notion runtime budgets (env)

These env vars are read at runtime by the Notion IO layer.

| Env var | Meaning | Default |
|---|---|---:|
| `CEO_NOTION_MAX_CALLS` | Max number of Notion HTTP calls allowed within a single read operation scope (budget context). | 2 |
| `CEO_NOTION_MAX_LATENCY_MS` | Max elapsed time budget for Notion read operation scope (soft-fail when exceeded). Used as global fallback for CEO snapshot when per-DB overrides are not set. | 1500 (generic reads), 4000 (CEO snapshot fallback if unset) |
| `CEO_NOTION_MAX_LATENCY_MS_TASKS` | CEO snapshot per-DB override for `tasks` (takes precedence over `CEO_NOTION_MAX_LATENCY_MS`). | (inherits global/fallback) |
| `CEO_NOTION_MAX_LATENCY_MS_PROJECTS` | CEO snapshot per-DB override for `projects` (takes precedence over `CEO_NOTION_MAX_LATENCY_MS`). | (inherits global/fallback) |
| `CEO_NOTION_MAX_LATENCY_MS_GOALS` | CEO snapshot per-DB override for `goals` (takes precedence over `CEO_NOTION_MAX_LATENCY_MS`). | (inherits global/fallback) |

Recommended values for CEO snapshot latency budgets: `3000`–`4000`.
